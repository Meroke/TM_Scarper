# Tmall Product Exporter Demo

这是一个最快可运行的 Chrome 浏览器插件 demo，用来读取当前天猫商品页，并导出 `.xlsx` 文件。

## 功能

- 读取当前选中的天猫商品页。
- 在天猫页面右上角显示粉色「采集」按钮。
- 通过「已采集」面板查看本地采集列表。
- 支持多次采集，并对采集列表批量选中、删除、复制、导出。
- 自动提取首图、商品名称、颜色分类、单价、网页链接。
- 手动填写数量和备注。
- 自动计算总价。
- 导出字段：首图、商品名称、颜色分类、单价、数量、总价、网页链接、备注。
- 下载首图并嵌入 Excel 单元格，不依赖远程图片链接。
- 表头高亮，内容区域添加细边框。
- 网页链接列写入 Excel 原生超链接，可直接点击打开。

## 安装运行

1. 打开 Chrome，进入 `chrome://extensions/`。
2. 打开右上角「开发者模式」。
3. 点击「加载已解压的扩展程序」。
4. 选择当前文件夹：`/mnt/hgfs/05code/TM_Scraper`。
5. 打开一个天猫商品详情页。
6. 页面右上角会显示粉色「采集」按钮。
7. 点击「采集」保存当前商品。
8. 点击「已采集」打开采集列表，可批量选中、删除或导出。

![用户界面](./用户界面.png)

浏览器扩展工具栏中的 popup 仍保留单页读取和单项导出能力。

## 文件结构

```text
TM_Scraper/
├── manifest.json           # Chrome MV3 插件配置、权限和脚本加载顺序
├── content.js              # 内容脚本启动入口，初始化页面控件并注册消息监听
├── product-extractor.js    # 天猫商品信息提取：首图、名称、颜色分类、价格、链接
├── collection-store.js     # 已采集商品的本地存储、数量和总价计算、规格清洗
├── collector-widget.js     # 页面右上角采集按钮、已采集列表和交互逻辑
├── clipboard.js            # 复制选中商品、复制单张首图和页面内下载辅助
├── excel.js                # 生成带图片、边框、超链接的 `.xlsx` 工作簿
├── popup.html              # 浏览器工具栏 popup 页面
├── popup.js                # popup 单项读取和导出逻辑
├── styles.css              # popup 样式
└── README.md               # 使用说明和项目结构
```

`manifest.json` 中的 `content_scripts.js` 按顺序加载这些模块。当前不使用构建工具，各模块通过 `window.TmallExporter` 共享少量接口。

## Demo 取舍

为了最快完成 demo，当前直接在浏览器插件里生成最小 `.xlsx` 文件，不依赖 Node、npm、构建工具或第三方库。

首图会先下载并转换为 PNG 后嵌入工作簿。若图片域名权限或图片下载失败，对应行会导出无图片版本并保留其他字段。

采集数据保存在 `chrome.storage.local`，刷新页面后仍可继续查看和导出。

天猫页面结构会变化，当前 `product-extractor.js` 使用多组选择器和页面 meta 信息兜底。真实使用前，需要用几个目标商品页测试并微调选择器。
